Arduino Node

Ever since the first Arduino Gateway, there was the idea to build a "network" of wireless nodes so that the Arduino could communicate on a higher level with the Raspberry master and its slaves and send sensor information etc. to the host.

The RadioHead library provides reliable, addressed but also unreliable communication over the air on a number of frequencies depending on the transceiver used. This means that we can use more or less teh same code regardless of the underlying hardware transceiver technology used.

So what would be the purposed of having such a network in the house

  • Send sensor data in a reliable and standard way over the air to the LamPI-node daemon
  • Act as an intermediate "node" in the network and extensing its range in the mesh topology
