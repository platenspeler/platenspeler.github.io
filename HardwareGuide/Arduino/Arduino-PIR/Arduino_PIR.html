Arduino PIR Sensor

One of the sensors I did not build yet is the PIR sensor: Waiting for a movement in a room send out an alarm to the master. I found a good sensor. However, it seems word best at 5V-20Vdc power supply. This is a pity, as it potenetially means we'll have to use more batteries or batteries with a higher voltage level to make the sensor work reliably. We have to look at this closer.

 

 

Sensor Hardware Description

  • It is great for microcontrollers such as arduino or funduino and many more including the rasberry pi! 
  • It is a HC--SR501 Sensor Module 
  • With an operating voltage of 5v to 20v
  • Operating voltage. It has a high value of 3.3 volts and a low of 0 (it is either on or off). It's delay time can be adjusted from 5 to 200seconds as well as the sensing range from 3meter to 7meters. The entire sensor with the board is 32mm by 24mm (standard pir motion sensor). The sensor either gives off a 3.3volt high value when motion is detected and then resumes or continues to give off a 0 voltage when no motion is detected. If you use an arduino or other micro controllers you can use the 3.3 volt out to power the sensor or use the 5volt power pin (vcc) to power the sensor. Eitther one will work.
  • Working voltage range: DC 4.5-20V
  • Static current: <50uA
  • High level output: 3.3 V / low 0V
  • Trigger mode: L unrepeatable trigger /H repeated trigger (Default Repeating trigger)
  • Delay time: 0.5-200S (adjustable) can be produced range 0.1 seconds - a few minutes
  • Blocking time: 2.5S (the default) can be produced range 0.1 seconds, tens of seconds
  • Size: 32mm*25mm circuit board
  • Induction angle: <100 degree cone angle
  • Working temperature: -15-+70 degrees
  • Induction of lens size: diameter: 23mm (default)

 

Sensor Hardware Setting

There are two wheels on the sensor that can be used to set the sensor sensitivity and the timeout value. For the timeout I use the leftmost value, which sets the value to around 5 seconds,
